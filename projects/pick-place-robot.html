Pick-and-Place Robot:
Kinematic Design & MATLAB Control.

A 3-DOF manipulator that senses its environment, derives joint angles from first-principles inverse kinematics, and executes a six-step autonomous transfer sequence — with no hard-coded positions.

01 · Problem Statement

Sense first. Compute. Then move.

Design and program a 3-DOF robotic manipulator to autonomously pick and place balls between three stations (A, B, C) at unknown and variable heights. The robot must first sense its environment, derive the correct joint angles from the measured geometry, and execute a predefined six-step transfer sequence — all without hard-coded position values.

The central engineering challenge: no pre-built kinematics solver could be used. Joint angles had to be derived from first principles, starting from the physical geometry of the link chain and the measured end-effector height Z.

→ Verified result: the robot completed all 6 sequential pick-and-place operations (B→C, C→A, A→B, B→A, A→C, C→B) in autonomous mode. No position was hard-coded — every joint angle was computed live from the derived IK equation.

03 · First-Principles Derivation

From geometry to joint angles.

Joint 1 — base rotation is direct: θ₁ = tan⁻¹(y / x). The three stations sit at fixed azimuths A = −90°, B = 0° (home), C = +90°, and the base motor converts encoder counts = 3.3 × θ₁ via the gear ratio. Joint 2 — arm inclination — is where the real derivation lives:

Height budget along the link chain
Working bottom-up, the total vertical reach to target height Z must satisfy the geometric chain constraint, with trigonometric constraint on the actuated arm L₃:
Z = L₀ + L₁ + L₂·sin(45°) + p  →  p = Z − L₂·(√2/2) − L₁ − L₀
sin(θ₂ − 45°) = (L₄ + p) / L₃
Substitute p and include the claw offset O = 55 mm
The offset was measured physically to prevent gripper collision with the station plane — a safety correction identified during hardware testing:
sin(θ₂ − 45°) = [L₄ − O + Z − L₂·(√2/2) − L₁ − L₀] / L₃
Consolidate constants
With numerical values (all in mm):
K = L₂·(√2/2) + L₁ + L₀ − L₄ + O
K = 67.17 + 50 + 70 − 110 + 55 = 132.17 mm
Final IK equation
Multiplied by 4 to convert joint angle to motor encoder counts via the empirically calibrated transmission ratio:
θ₂ = [ sin⁻¹( (Z·1000 − 132.17) / 185 ) + 45° ] × 4
Z in meters (×1000 converts to mm). Implemented directly in MATLAB as asind((l4−O+Z*1000−l2*sind(45)−l1−l0)/l3)+45)*4
calc_inv() — IK function as implemented 
function theta_2 = calc_inv(Z)        % inverse kinematics
    O_ffset = 55; l0 = 70; l1 = 50;     % link & offset constants [mm]
    l2 = 95; l3 = 185; l4 = 110;
    theta_2 = (asind((l4 - O_ffset + Z*1000 ...
              - l2*sind(45) - l1 - l0)/l3) + 45) * 4;
end

04 · Key Design Decision

Adaptive height sensing.

Rather than hard-coding station heights, the system performs an autonomous environment scan at startup: the base rotates to each station, the ultrasonic sensor reads Z, and the IK equation computes θ₂ dynamically. The same code handles any table-height variation or station repositioning without recalibration — sensor-driven first-principles computation driving real hardware.

Adaptive height-sensing loop — no hard-coded positions 
% Inspect environment: measure each station height before any pick/place
home(t_sens1,base,t_sens2,incl,claw);   % return to reference
locate(base, a(1));    a(2) = readdis(sonicsens);  % station A height Z_A
home(t_sens1,base,t_sens2,incl,claw);   b(2) = readdis(sonicsens);  % station B height Z_B
locate(base, c(1));    c(2) = readdis(sonicsens);  % station C height Z_C

% Per operation: compute arm angle from MEASURED height via IK
theta2   = calc_inv(a(2));    % pick angle  = f(measured Z)
theta2_f = calc_inv(c(2));    % place angle = f(measured Z)
pick(incl,claw,t_sens2,theta2);
place(incl,claw,t_sens2,theta2_f);

Reading the diagram: the elevation cross-section shows the arm at an intermediate angle. The dashed horizontal line is the reference plane at the top of L₃'s sweep.

p + L₂·(√2/2) + L₁ + L₀ = Z
sin(θ₂ − 45°) = (L₄ + p) / L₃

The 45° offset in the sine term arises from the fixed L₁–L₂ angle of 135° (complement = 45°), visible as the angled base of the triangular link configuration.

The claw offset O = 55 mm (physically measured) was subtracted from L₄ to prevent the gripper from driving into the station surface — a safety-critical correction found during hardware testing.

06 · Engineering Approach

What I built.

  • Derived the complete inverse kinematic model from geometric first principles (steps 01–04 above)
  • Identified and incorporated the claw offset (O = 55 mm) by physical measurement to prevent end-effector collision
  • Implemented the full MATLAB control algorithm (FINAL.m): homing, height-sensing loop, IK call, pick/place sequencing
  • Designed the adaptive sensing architecture — dynamically computing θ₂ per measured Z before each operation sequence
